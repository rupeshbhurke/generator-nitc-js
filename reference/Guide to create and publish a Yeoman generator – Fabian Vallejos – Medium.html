Guide to create and publish a Yeoman generator

Fabian Vallejos
Jun 30, 2017

Follow this step-by-step guide to easily create and publish a simple Yeoman generator for Angular JS components in 6 steps.

Yeoman is a web’s scaffolding tool for web apps. It helps us to stay productive by simplifying some of the repetitive tasks we do as developers.

Requirements

We need to install NodeJS with npm, and Yeoman. To download and install NodeJS/npm, please visit https://nodejs.org/ and follow the instructions.

To install yeoman, visit http://yeoman.io/

We also need a github repository, and a npmjs.org account. Make sure you have both. Visit https://github.com/ and https://www.npmjs.com/ to create accounts if you don’t have one yet.

Step #1: Create a github repository for our generator

Go to your github account and create a new repository.

Please note that the repository name must start with generator-, and be followed by a unique name to npm. This is required to publish our generator.

Enter a unique name and a description. The name must be unique, like generator-your-unique-name.

Click on Create repository when done. You can initialize the repository with a README file if you like. In this guide we’re going to add all the files manually. The name has to be unique. You can check whether the name exists or not by using the following url: https://www.npmjs.com/package/generator-s9s-ngcomponent (change generator-s9s-ngcomponent by the name of your choice). If the repository doesn’t exist, you will see a message saying that the name is available and can belong to you.

Copy the repository URL to use in the next step.

Step #2: Initialize our repository

Let’s clone our repository to work locally. Open a terminal and clone the repository you have created. Example:

$ git clone git@github.com:vallejos/generator-s9s-ngcomponent.git
Cloning into 'generator-s9s-ngcomponent'...
warning: You appear to have cloned an empty repository.

Use the copied URL from the previous step. We can ignore the warning about cloning an empty repository since we are going to create files soon and commit the changes when we’re done testing the generator locally.

Change the working directory to the newly cloned folder:

$ cd generator-s9s-ngcomponent

Let’s initialize npm:

$ npm init

Let’s go step-by-step on each option. First, enter a name. Press enter for the default name:

Now enter a version number:

Next, a description for the generator:

description: S9S yeoman generator for Angular components

For the next options, leave the entry point as the default value (index.js), the test command (empty), and git repository.

entry point: (index.js) <press enter>
test command: <press enter>
git repository: (https://github.com/vallejos/generator-s9s-ngcomponent.git) <press enter>

The next step is important, in the keywords section, we need to add some keywords to make our generator searchable. One of the keywords must be yeoman-generator. Yeoman uses this keyword to find generators. We can also add other keywords:

keywords: yeoman-generator, angular, component, javascript, builder, scaffold, generator, s9s

Finally, add information for the author and a license.

author: Fabian Vallejos <vallejosfab@gmail.com>
license: (ISC) MIT

Now it’s time to add some files.

Step #3: Add files

A generator is basically a NodeJS module. It is deeply linked to the file system and the directory tree. When you call your generator, a default generator called app is executed. Each sub-generator is contained inside a folder with its own name. For example, when you run the app sub-generator, yeoman will look for the files inside an app/ folder. In our example, we’re not going to use any sub-generators, so we’re going to create an app/ folder and put the files required inside.

Let’s create the folder and a first required file:

$ mkdir app
$ touch app/index.js

In the index.js file we’re going to initialize our NodeJS module. This module will be run by default when we call our generator.

We still need to do something important. We need to tell NodeJS where to find our generator files. Open your package.json file, and add the following lines:

"files": [
  "app"
]

Our package.json file should look like this now:

We’re not going to go into much details about the specific functionality of our example generator. Let’s start to add a basic constructor now. Open your app/index.js file and copy/paste the following:

'use strict';
const Generator = require('yeoman-generator');
module.exports = class extends Generator {
  constructor(args, opts) {
    super(args, opts);
    this.log('Initializing...');
  }
  start() {
    this.log('Do something...');
  }
};

Save the file. This is the most basic constructor, it will just display a log message to the terminal. We will see this in the next step. Note that besides the constructor, I also added a start() method. If you don’t add at least one method, you will get an error when you want to test it. This is quite simple, the module is initialized by the constructor, and then the start() method is executed. If we add more methods, they are all executed one after the other.

Please refer to http://yeoman.io/authoring/index.html for more information about sub-generators, constructors, and much more.

Step #5: User interaction and templates

Of course before publishing our generator, we want to make something extra. Let’s do two basic things now:

  1. Do some user interaction
  2. Copy a file

There’s much more that we can do with our generator, but that’s a topic for another post.

Let’s start by asking the user to enter a name for our module. Open the app/index.js file and replace the contents of the start() method by adding the following:

this.prompt([
    {
      type    : 'input',
      name    : 'name',
      message : 'Enter a name for the new component (i.e.: myNewComponent): '
    }
  ]).then( (answers) => {
    // create destination folder
    this.destinationRoot(answers.name);
  });

Here we do two things, first, we prompt the user to enter a name, and then, we create a folder with that name. Save the file and run it:

$ yo s9s-ngcomponent
Initializing...
? Enter a name for the new component (i.e.: myNewComponent):  testMe

Now, do a ls and see if a folder named testMe has been created. It should be there!

Now, let’s copy a file, an html template for example. We need to create a folder in the app/ folder to save our source files that will be copied to the destination folder.

$ mkdir app/templates
$ touch app/templates/index.html

Now, we need to copy these files. Add the following in the index.js file, after the this.destinationRoot(answers.name); line:

this.fs.copyTpl(
  this.templatePath('index.html'),
  this.destinationPath(answers.name + '.html')
);

The previous will search for a file named index.html inside the default templates folder, and copy it to the destination folder. At the same time, we’re changing the name of the file to be the same as the component name.

If we run this again, we should see something like:

$ yo s9s-ngcomponent
Initializing...
? Enter a name for the new component (i.e.: myNewComponent):  testMe
   create testMe.html

If we list the contents of the testMe folder, we should see our new file there. We can do a lot more, let’s do some extra thing now. Since we’re copying a template, it would be nice if we can insert/change contents of the file, right? We can do that using EJS (Effective JavaScript. See http://ejs.co/).

Open the app/templates/index.html file and add the following:

<div>
  <p><%= message %></p>
</div>

Using EJS, we’re going to replace the message tag by something else. It could be anything, but let’s do the classic hello world :)

In the app/index.js file, pass an extra parameter to the copyTpl method:

{ message: 'Hello world!' }

It should look like this:

this.fs.copyTpl(
  this.templatePath('index.html'),
  this.destinationPath(answers.name + '.html'),
  { message: 'Hello world!' }
);

Let’s run our generator again:

$ yo s9s-ngcomponent
Initializing...
? Enter a name for the new component (i.e.: myNewComponent):  testMe
   conflict testMe.html
? Overwrite testMe.html? overwrite
   force testMe.html

Note that since we’re using the same testMe folder, there’s a conflict because the testMe.html file already exists from our previous test. Let’s just overwrite it. When prompted, answer y and press enter to overwrite the html file.

Now, if we look at the content of the html file, we see that the message tag has been replaced by our hello world message.

$ cat testMe/testMe.html
<div>
  <p>Hello world!</p>
</div>

Our generator is now ready to be published.

Step #6: Publish

Let’s first commit the changes to github. Add the files, commit them, and push them to the repository (don’t forget to delete the testMe folder before doing this, or just add a .gitignore file to ignore it).

$ rm -rf testMe
$ git add .
$ git commit -S -a -m "S9S Angular Generator v0.0.1"
$ git push -u origin master

In the example I’m using, I have also added the files: README.md, LICENSE, .gitignore, as well as other templates I use for the generator-s9s-ngcomponent. You can check all those files in my github repository: https://github.com/vallejos/generator-s9s-ngcomponent

Now it’s time to publish the generator to npmjs.org

$ npm publish
+ generator-s9s-ngcomponent@0.0.1

It’s done! Our generator has been published and it’s ready to use. Please note that it may take up to an hour to see our generator published in the npmjs.org repository. You can double check your npmjs.org profile to see if it’s there, or just go to the package url. In our example: https://www.npmjs.com/package/generator-s9s-ngcomponent

This concludes our guide, but let’s add a couple more things that are important:

Guide to update your generator

If you need to update your generator, you will need to follow this procedure:

  1. Work on the updates
  2. Update the version number in the package.json.
  3. Commit and push the changes to github
  4. Run $ npm publish again to publish your updated generator.

Let’s say that we need to update our README.md file. Make any changes to an existing file, add new files, delete something, whatever. When you’re done, just commit and push your changes to github, and then run the npm publish command again. You should see your version bumped to the one you set in your package.json file.

Example:

$ npm publish
+ generator-s9s-ngcomponent@0.0.2

Using your generator

Once you have created your generator, it’s easy to use it. Just install it as you would do with any other yeoman generator. You can share your generator with your team or just use it yourself to speed up some of those mechanic daily processes.

Install it:

$ npm install -g generator-s9s-ngcomponent

Use it:

$ yo s9s-ngcomponent

Conclusions

In this step-by-step guide we have seen how to create and publish a simple Yeoman generator. The requirements are just:

  • Have a Github account to publish the repository
  • Have a npmjs.org account to publish the generator
  • Create a unique name that must start with generator-
  • The package must have a keyword yeoman-generator
  • Your NodeJS module must extend the class Generator

Yeoman allows us to do a lot more things, this is just a basic start to understand the basics. More information is available at http://yeoman.io/

For information about the generator created on this guide, please visit https://github.com/vallejos/generator-s9s-ngcomponent or https://www.npmjs.com/package/generator-s9s-ngcomponent
